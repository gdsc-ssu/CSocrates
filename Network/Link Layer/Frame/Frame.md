# 개요

Frame 영어 단어의 뜻은 어떤 것을 담아내는 틀이다.
데이터 통신에서의 Frame은 데이터와 제어 정보를 합친 것이라고 나와있다. 다양한 프로토콜들에 의해 교환되고 운반되는 데이터의 단위를 말하며, IEEE에서는 Frame보다는 [[PDU]]라는 용어로 사용하고 있음.

![](http://www.ktword.co.kr/img_data/849_1.JPG)

위 그림은 일반적인 프레임 구성 형태이며, 대부분의 제어 정보는 Header에 들어있다.

# 프레임의 경계

프레임의 시작과 끝은 어디서부터 어디까지일까? Frame의 경계를 짓는 것을 Framing이라고 한다. (프레이밍, 프레임 동기)
프레이밍을 하면서 비트 송/수신 동기화를 하기도 한다.

1. 집중 프레임 동기 (Bunched Frame Alignment)
	* 프레임 시작 및 끝에 독특한 비트열을 배치하여 프레임을 식별.
2. 분산 프레임 동기
	- 각 프레임에 일정한 시간 간격으로 프레임 비트를 삽입.
이외의 동기 방식은 생략.

# 프레임의 길이

프레임의 길이는 고정적일 수도(ATM 등), 가변적일 수도(Ethernet 등) 있다.
프레임 길이의 상한선은 [[MTU]] 라고 한다.

# 프레임의 종류

대표적으로 3가지가 있다.
- MAC 프레임
- HDLC 프레임
- [[Ethernet]] 프레임

## MAC 프레임

Media Access Control Frame

![](http://www.ktword.co.kr/img_data/1816_1.JPG)

Preamble과 SFD는 위에서 얘기 했던 프레임 동기와 관련이 있다.
MAC 프레임 자체는 2계층이지만 앞에 1계층인 선행부(Preamble과 SFD)가 붙는다.
Preamble은 10101010이 7번 반복되는 형태이고, 
SFD는 10101011이다. 
즉, MAC 프레임에서 헤더 앞에 붙는 정보는 0xAAAAAAAAAAAAAAAB가 된다.

목적주소 (Destination MAC Address) : 데이터 프레임이 전송될 목적지 장비의 MAC 주소.
출발주소 (Destination MAC address) : 데이터 프레임을 전송한 장비의 MAC 주소.
길이/유형 필드 (Length/Type Field) : 데이터 프레임 내용의 종류와 길이. 일반적으로 이더넷에서는 프로토콜 유형 (IPv4, IPv6, ARP)를 나타내는 값이 들어감.
데이터 : 실제 데이터가 들어감. 상위 계층에서 잘 포장한 데이터가 이 곳에 들어감.
FCS(Frame Check Sequence) : 오류 검사 비트. 4바이트를 사용하며, CRC 방식을 사용.

## HDLC 프레임

High-level Data Link Control

고속 데이터 전송에 적합한 범용의 데이터 링크 전송의 제어 방식.

반이중, 전이중 방식을 모두 지원.

![](http://www.ktword.co.kr/img_data/411_1.jpg)

Flag에는 프레임 동기를 위해 01111100을 각각 넣어준다. 이는 0x7E이다.
I-frame은 사용자 정보 + 제어 정보를, S-frame은 제어 정보만, U-frame은 링크관리정보를 담는다.

## Ethernet 프레임

이더넷이 최초로 발표된 후에 회사마다 각각의 프레임 포멧을 발표했다. 표준화가 잘 안되어서 프레임 포멧이 약간씩 다르다.

![](http://www.ktword.co.kr/img_data/2965_1.jpg)


Preamble는 MAC 프레임에서 사용했던 것과 같이 10101010이 7번 반복되고 SFD는 10101011이다.

Padding은 Data가 46바이트를 넘기지 못했을 때 채우는 비트열이다. 이더넷 프레임의 최소 크기는 64바이트인데 (물리 계층 헤더 제외), 앞 뒤로 총 16바이트를 빼고 Data에서 46바이트가 있어야 한다.
왜 46바이트를 꼭 채워야 하는 것일까?
- 충돌 도메인을 관리하기 위해 CSMA/CD 프로토콜을 사용하는데, 충돌을 신뢰성 있게 탐지하고 최소 충돌 감지 시간을 확보하기 위해 최소 크기를 정해두었다.
