# 4주차 복습

## OSI 7 Layer, TCP/IP 4,5 Layer

OSI 7 Layer

-  Application Layer: 응용 프로세스가 OSI 환경에 액세스하는 유일한 수단 제공.
-  Presentation Layer: 응용 엔티티가 통신하는 정보의 표현 관리.
-  Session Layer: 데이터 교환 관리, 대화 조직화 및 동기화 제공.
-  Transport Layer: 신뢰성 있는 데이터 전송과 비용 효율적 전송 책임.
-  Network Layer: 데이터 전송 경로 설정 및 중계 고려 사항에서의 독립성 제공.
-  Data Link Layer: 데이터 링크 연결 및 데이터 유닛 전송 관리.
-  Physical Layer: 물리적 연결 활성화, 유지, 비활성화 및 비트 전송 책임.

TCP/IP Model

-  Application Layer: OSI의 프레젠테이션과 응용 계층 기능 결합.
-  Transport Layer: 종단간 통신 서비스 제공, TCP 및 UDP 포함.
-  Internet Layer: IP를 사용한 데이터 소스에서 목적지 호스트로의 전달, 종단간 전송 보장 없음.
-  Link Layer: 직접 연결된 네트워크 통신을 위한 프로토콜 구현.

## udp

-  UDP (사용자 데이터그램 프로토콜): 패킷 전송 속도가 빠르지만, 전달되는 패킷의 도착을 보장하지 않는 전송 계층 프로토콜. 패킷 손실이 발생할 수 있으나, 신속한 데이터 전송이 필요한 상황에 적합.
-  유튜브의 TCP 사용: 실시간 스트리밍이 아닌 녹화 파일 스트리밍이 주된 서비스이므로 TCP 사용. TCP를 통해 네트워크 상태에 따라 데이터 전송량을 조절하는 DASH 방식 채택.
-  UDP와 DDOS 공격: UDP는 응답 확인 없이 무지성 송수신이 가능하여 DDOS 공격에 취약. 서버 리소스 과부하나 포트 포화로 인해 서버 다운 가능.
-  브로드캐스트와 멀티캐스트: TCP는 연결 설정에 핸드 쉐이크 과정 필요로 여러 대상에게 동시에 데이터 전송 불가. 반면, UDP는 연결 설정 없이 브로드캐스트(전체 네트워크 대상 전송)와 멀티캐스트(특정 그룹 대상 전송)에 적합.

## tcp, udp header

TCP Header

-  Source/Destination Port Number: 16비트, 소켓 주소 생성에 사용.
-  Sequence Number: 32비트, 패킷 순서 부여 및 신뢰성, 흐름 제어 기능 지원.
-  Acknowledgement Number: 32비트, 다음 기대 Sequence Number 지정.
-  Header Length: 4비트, 20~60바이트 헤더 길이 지정.
-  Flag bits: 6개 비트 (CWR, ECE, URG, ACK, PSH, RST, SYN, FIN)로 구성, 통신 제어 목적.
-  Window Size: 16비트, 혼잡 제어 및 흐름 제어 사용.
-  Checksum: 16비트, 헤더 체크섬.
-  Urgent pointer: 16비트, 긴급 데이터의 마지막 바이트 위치 지정.

UDP Header

-  Source/Destination Port Number: 16비트, 발신지 및 목적지 포트 번호 지정.
-  Packet Length: 최소 8바이트, 최대 65,535바이트. UDP 헤더를 포함한 전체 패킷 길이를 바이트 단위로 표시.
-  Checksum: 선택적 사용. 0일 경우 체크섬 계산하지 않음.

## 세션계층 톺아보기

세션 계층

-  어플리케이션 관련 층, 프로세스 간 통신 프로토콜 관리.
-  "세션" 관리: 클라이언트와 서버 간의 연결 관리.
-  단위: 메시지.

세션의 정의

-  둘 이상의 통신 장치 또는 사용자 간의 지속적이고 상호작용적인 정보 교환.
-  네트워크 상 클라이언트와 서버 간의 연결.

세션 계층 vs 전송 계층

-  세션 계층: 데이터 송수신 중단 시 연결 복구 후 중단된 지점부터 전송 재개.
-  전송 계층: 데이터 전송 중단 시 처음부터 재전송.

세션계층의 역할

-  세션 연결, 제어, 해제.
-  동기화: 오류 복구를 위한 동기점 사용.
-  대화 기능: 데이터 전송 구조화 및 흐름 관리.
-  토큰: 대화 관리를 위한 특수 메시지.
-  NetBIOS의 역할

네트워크 상의 컴퓨터 간 세션 및 통신 관리.

-  세션 설정, 유지, 종료 관리.
-  세션의 종류
